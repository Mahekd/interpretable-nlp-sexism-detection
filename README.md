# interpretable-nlp-sexism-detection

MSc capstone project — interpretable NLP models for online sexism classification on the SemEval-2023 EDOS benchmark, using fine-tuned BERT and RoBERTa with LIME (primary) and SHAP (comparative validation + bias audit) for explainability.

## Repo structure

```
.
├── data/                          # EDOS dataset (labelled CSVs tracked in git; large unlabelled corpora gitignored)
│   ├── edos_labelled_aggregated.csv
│   └── edos_labelled_individual_annotations.csv
├── notebooks/
│   ├── 01_eda_preprocessing.ipynb # exploratory data analysis
│   ├── 02_training.ipynb          # fine-tunes all 6 classifiers (BERT/RoBERTa × Tasks A/B/C)
│   ├── 03_explainability.ipynb    # LIME + SHAP + faithfulness metrics + Task C bias audit
│   └── 04_results_summary.ipynb   # consolidates results.json into a comparison table
├── src/
│   ├── data.py                    # loading, task framing (A/B/C), class weights, tokenized Dataset
│   ├── augment.py                 # optional EDA / back-translation augmentation (train split only)
│   ├── train.py                   # fine-tuning CLI (works with any AutoModelForSequenceClassification)
│   └── explain.py                 # LIME/SHAP wrapper, faithfulness metrics, gender-bias audit
├── outputs/                       # trained checkpoints + results.json (gitignored, regenerated by training)
├── requirements.txt
└── .gitignore
```

## Setup

```bash
pip install -r requirements.txt
```

Everything is designed to run either locally/Claude Code (for writing and debugging code) or in Google Colab (for GPU-backed training and explanation generation) — the notebooks default to cloning this repo fresh, so no manual file copying is needed either way.

## Pipeline: what to run, where, and in what order

| Step | File | Where to run it | Needs GPU? |
|---|---|---|---|
| 1. EDA | `notebooks/01_eda_preprocessing.ipynb` (Section 1) | Colab or local Jupyter | No |
| 2. Preprocessing (demo) | `notebooks/01_eda_preprocessing.ipynb` (Section 2) → `src/data.py` | Colab or local Jupyter | No |
| 3. Augmentation (optional, preview) | `notebooks/01_eda_preprocessing.ipynb` (Section 3) → `src/augment.py` | Colab or local Jupyter | No |
| 4. Training | `notebooks/02_training.ipynb` → calls `src/train.py` | Colab (GPU runtime) | Yes |
| 5. Explainability | `notebooks/03_explainability.ipynb` → calls `src/explain.py` | Colab (GPU runtime) | Yes |
| 6. Results summary | `notebooks/04_results_summary.ipynb` | Colab or local Jupyter | No |

### 1. EDA — `01_eda_preprocessing.ipynb`, Section 1
Class distributions, text length stats, label imbalance across Tasks A/B/C. Purely exploratory — no GPU needed, run interactively wherever you can view plots.

### 2. Preprocessing — `01_eda_preprocessing.ipynb`, Section 2 (backed by `src/data.py`)
`src/data.py` isn't a script you run on its own — it's imported by `train.py` and `explain.py` to load the CSV, slice it into Task A/B/C label sets (`build_task_frame`), apply the official train/dev/test split (`get_splits`), compute class weights, and tokenize on the fly (`SexismDataset`). Section 2 of the EDA notebook demonstrates this pipeline directly: it clones the repo, runs `build_task_frame`, and prints a tokenized example so you can see exactly what the model receives — case preserved, `[URL]`/`[USER]` kept as-is, no lemmatization or stopword removal.

### 3. Augmentation preview — `01_eda_preprocessing.ipynb`, Section 3 (backed by `src/augment.py`)
Off by default in `src/train.py` (`--augment none`). This section runs `eda_augment` on 5 minority-class Task C examples and prints original-vs-synthetic pairs side by side, so you can inspect what it actually produces before deciding to use it for real, and have concrete examples for the write-up's justification. The full augmentation (applied to the whole train split, not just this preview) happens inside `02_training.ipynb`.

### 4. Training — `02_training.ipynb` (runs `src/train.py`)
Fine-tunes all 6 required classifiers: BERT and RoBERTa on each of Tasks A, B, C. Open in Colab with a GPU runtime, run top to bottom. Each run saves a checkpoint to `outputs/best_model_task{A,B,C}_<model>/` (the model name is part of the folder, so RoBERTa and BERT runs for the same task don't overwrite each other) and a `results.json` with the hyperparameters used plus dev/test macro-F1 and the full classification report. Optional hyperparameter-sweep runs (`--run_name`) get their own sub-folder too, e.g. `outputs/best_model_taskA_roberta-base_lr3e-5/`.

Command form, if running manually instead of via the notebook:
```bash
python -m src.train --task A --model_name roberta-base
python -m src.train --task A --model_name bert-base-uncased
python -m src.train --task B --model_name roberta-base
python -m src.train --task B --model_name bert-base-uncased
python -m src.train --task C --model_name roberta-base
python -m src.train --task C --model_name bert-base-uncased
```

Augmentation (`src/augment.py`) is off by default, matching the proposal's methodology (class weighting only). If Task B/C performance needs it later, add `--augment eda` to the relevant runs — no code changes required.

Colab sessions are ephemeral — the notebook's last cell backs up `outputs/` to Google Drive. Do this before the runtime disconnects.

### 5. Explainability — `03_explainability.ipynb` (runs `src/explain.py`)
Run after training, using the checkpoints in `outputs/`. Produces LIME explanations per prediction, SHAP comparative attributions, sufficiency/comprehensiveness faithfulness scores, and the Task C gender-neutral-term bias audit. This is interactive (visualizations render best in a notebook), so it's not meant to be a pure CLI run, though `python -m src.explain --task A --checkpoint outputs/best_model_taskA_roberta-base` works for a quick text-only check.

### 6. Results summary — `04_results_summary.ipynb`
Reads every `outputs/best_model_task*/results.json` and builds one comparison table (BERT vs RoBERTa, across Tasks A/B/C), plus lines it up against the published EDOS baselines (Most Frequent, DistilBERT, DeBERTa-v3, Mahmoudi, Goldzycher, Best SemEval System) for the write-up.

## Notes

- The official train/dev/test split is used unmodified (14000/2000/4000 rows), so results are directly comparable to the SemEval-2023 leaderboard.
- The two ~90MB unlabelled corpora (`gab_1M_unlabelled.csv`, `reddit_1M_unlabelled.csv`) are gitignored — not used by this pipeline, and available from the [official EDOS repo](https://github.com/rewire-online/edos) if ever needed.
- Research papers referenced in the proposal are kept locally, not committed (copyrighted third-party work).
